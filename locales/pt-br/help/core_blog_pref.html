Detalhes do blog

ID do Blog
ID único do blog. Obrigatório, só pode ser modificado por um super administrador.
Nome do blog
Name of the blog, mandatory.
URL do Blog
Complete blog URL. Mandatory, can only be modified by a super administrator.
Blog status
Define como a URL é lida. PATH_INFO é recomendado, QUERY_STRING deve ser usado se PATH_INFO não funcionar. Usando o modo PATH_INFO, a URL do blog deve terminar com "/" e usando QUERY_STRING, ele deve terminar com um "?".
Blog description
  • On-line: Blog aberto aos visitantes.
  • Off-line: Blog fechado para os visitantes, mas aberto a escritores.
  • Removido: Blog fechado para visitantes e escritores, aberto apenas a super administradores.
Descrição do blog
Aqui você pode colocar qualquer descrição que você quer para o seu blog. Este é um texto simples, sem qualquer formatação.

Configuração do blog

Nome do editor do blog
Name of the person responsible for the blog. It may be the owner or the publisher if he exists.
Copyright notice
Note about the reproduction rights for the blog.
Default language
Language of the blog interface. Either this language is among the avalaible ones, and the interface will be translated, or it is not, and the interface will then default to English.
Blog timezone
Defines the timezone that will be used for comments and trackbacks.
Date formats
The date format of the blog posts. See the date and time format reference section below for further details.
Time format
The time format of the blog posts. See the date and time format reference section below for further details.
Display .. entries per page
The number will be used as the limit for the number of entries on the homepage, the first page of each category and the rss feeds.

Comentários e trackbacks

Aceitar os comentários
Globalmente aceita comentários no blog. Toma precedência sobre a configuração por postal.
Moderar os comentários
Com esta opção, os comentários serão publicados apenas após a aprovação do editor.
Deixar os comentários abertos para X dias
O número de dias durante o wich comentários será aberto após a publicação do post. Quando vazio, os comentários serão mantidos abertos indefinidamente.
Sintaxe wiki para os comentários
Permite que um subconjunto da sintaxe wiki seja usado nos comentários.
Aceitar os trackbacks
Globalmente aceita trackbacks no blog. Toma precedência sobre a configuração de cada post.
Moderar os trackbacks
Com esta opção, os rastreadores serão publicados somente após o acordo do editor.
Deixar as trackbacks abertas por X dias
O número de dias durante o wich trackbacks será aberto após a publicação da publicação. Quando vazio, os trackbacks serão mantidos abertos indefinidamente.
Acrescentar a relação "nofollow" nos links de comentários e nos trackbacks
Adiciona um atributo em links em comentários e trackbacks, instruindo os bots de pesquisa para não segui-los. Esses recursos antispam ainda não provaram seu valor.

Apresentação do blog

Formato das datas
Define o formato da data para os posts do blog. Veja a seção "Formatar referência" abaixo para mais informações. Você pode usar a lista suspensa para selecionar rapidamente um padrão pré-existente.
Formato das horas
Define o formato de hora para os posts do blog. Veja a seção "Formatar referência" abaixo para mais informações. Você pode usar a lista suspensa para selecionar rapidamente um padrão pré-existente.
Exibir emoticons nos posts e nos comentários
Transforma algumas strings como :-) ou ;-) em imagens.
Exibir X postagens por página
Irá exibir as entradas X na página principal do blog, bem como nas diferentes categorias.
Exibir entradas X por feed
Irá exibir as entradas X nos feeds do blog.
Exibir comentários X por feed
Irá exibir comentários em X nos feeds de comentários do blog.
Truncar feeds
Esta opção só fornecerá um resumo dos posts no feeds.
Incluir subcategorias na página de categoria e categoria posts feed
Quando esta opção é ativada, postagens em sub-categorias serão exibidas nas páginas de categorias pai bem como nos seus feeds.

Mídia e imagens

Tamanhos de imagens gerados (em pixels)
Quando você adiciona uma imagem no gerenciador de mídia, outros tamanhos são gerados automaticamente. Aqui você pode definir a dimensão do tamanho mais longo (largura ou heigth) de cada tipo de imagens geradas. Se deixar um campo vazio, o gerenciador de mídia não criará o miniatura correspondente.
Atributos de inserção de imagem padrão
As configurações selecionadas serão aplicadas em todas as novas imagens adicionadas ao gerenciador de mídia.
Título da imagem inserida
Selecione o formato das tags "title" aplicadas às suas imagens, com base nas informações obtidas dos metadados das imagens.
Usar data original de mídia, se possível
Quando esta opção é ativada, se os metadados da imagem contiverem uma data, ela será usada. Se não, a data de envio será usada.
Tamanho da imagem inserida
Define o tamanho padrão usado ao inserir uma imagem em um post.
Alinhamento da imagem
Define o alinhamento padrão usado ao inserir uma imagem em um post.
Inserir um link na imagem original
Quando esta opção estiver ativada, as imagens inseridas em um post de blog serão clicáveis e apontarão para a imagem do tamanho original.

Blog details

Alguns campos podem ser configurados para formatar a data e hora para suas necessidades. Aqui está uma referência:

  • %a: abreviated week day (local).
  • %A: complete week day (local).
  • %b: abbreviated month name (local).
  • %B: complete month name (local).
  • %c: default local date and time display.
  • %C: century (the year, divided by 100 and round up beetween 00 and 99)
  • %d: numeric month day (from 01 to 31)
  • %D: identical to %m/%d/%y
  • %e: numeric month day (from 1 to 31)
  • %g: identical to %G, 2 digits.
  • %G: The year on 4 digits, according to the week number (cf. %V). Format and value identical to %Y, except that if the number of the week belongs to the preceding or following year, the current year will be used instead.
  • %h: identical to %b
  • %H: digital hour of the day, 24H mode (from 00 to 23)
  • %I: digital hour of the day, 12H mode (from 01 to 12)
  • %j: day of the year, digital (from 001 to 366)
  • %m: month, digital (from 1 à 12)
  • %M: minute, digital
  • %n: newline character
  • %p: either `am' or `pm', according to the absolute time, or depending of the local settings.
  • %r: the hour, a.m. and p.m. format
  • %R: the hour, 24h format
  • %S: seconds, digital
  • %t: tab character
  • %T: the current time (identical to %H:%M:%S)
  • %u: day week, digit, from 1 to 7. (1 stands for monday)
  • %U: week number, the first sundqy of the year being the first day of the first week.
  • %V: the week numpber as defined in ISO 8601:1988, digital, from 01 to 53. The week 1 is the first week with more than 4 days in the current year, whose monday is the first day. (Use %G or %g for the year elements corresponding to the number of the week for the considered timestamp.)
  • %W: number of the week in the year, the first monday of the year being the start of the first week
  • %w: day of the week, digital, 0 stands for sunday
  • %x: prefered format for the date without the time
  • %X: prefered format for the time without the date
  • %y: the year, two digits (from 00 to 99)
  • %Y: the year, four digits
  • %Z ou %z: timezone, name or abbreviation
  • : a litteral `%' character
